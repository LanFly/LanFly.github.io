个性化

LanEditor的一个特点之一是有关键字提示功能，尽管这个功能在一个在线编辑器中出现还是比较少的，也有很多人不理解这有什么用。不过我当初做这个编辑器就是为了符合自己的习惯，所以我个人觉得它还是有必要的。

我写这个文档的时候，或是写一些技术类文章的时候，不可避免的要写些代码。如果是写一段CSS，大部分都是重复的，我可不会一个一个字敲上去。js中有很多函数、关键字，难道也要一个一个敲？还有一些是不太熟悉的还要临时查一下有没有写错名字。

这就是理由吗？

这篇文档你有没有发现，出现很多次LanEditor。写一次两次倒没啥，问题是这至少几十次了 ::>_<:: 写起来要命滴！中间还有个大写字母呢(+﹏+)~狂晕

幸好我不是一个一个敲上去的，也不是复制粘贴的。而是关键字提示的。只要输入lan按回车就插入了~(≧▽≦)/~啦啦啦

什么？这是哪门子语言的关键字？怎么会提示！逗我。

这就是关键字扩展的功能了！

关键字扩展

关键字扩展允许你在默认的关键字库下随意扩展个人词库。例如我的：

LanEditor.Keyword.extend({
    "lan": [
        "LanEditor",
        "API",
        "GetRenderHTML",
        "PluginsMode",
        "OpenAutoSymbol"
    ],
    "contact": [
        "bluescode@outlook.com",
        "1260022720"
    ]
});

通过Keyword模块的extend方法，你可以随意扩展个人词库。

extend的参数任何时候都应该是一个数组对象。否则它不会接受。它可以无限次调用，无限次扩展，只要你喜欢。扩展的字库会以追加的方式加入查找列表。

数组的名字由你自己定义，它不会影响最终查找的结果，只是用来分类。在以后的版本中，它可能会根据分类从而影响插入的结果，为了升级做准备，现在不用担心。

数组里面就是你想要加入查找列表的清单了，它允许出现重复值，如果有多个，也会被查找出来。建议是规范扩展，不要出现重复的。

扩展有多种写法。你可以根据自己的实际情况选择。这里给出一些例子让你更好的理解扩展的原理。

1.同一分类可以分开扩展。下面的效果和上面的是相同的

LanEditor.Keyword.extend({
    "lan": [
        "LanEditor",
        "API",
    ],
    "lan": [
        "GetRenderHTML",
        "PluginsMode",
        "OpenAutoSymbol"
    ]
});

2.也可以一次扩展多个分类

LanEditor.Keyword.extend({
    "lan": [
        "LanEditor",
        "API",
    ],
    "functions": [
        "GetRenderHTML()",
        "GetFileList()",
        "NewFile(filename)",
        "GetFileContent(filename)"
    ]
});

3.还可以分次扩展

LanEditor.Keyword.extend({
    "lan": [
        "LanEditor",
        "API",
    ]
});

LanEditor.Keyword.extend({
    "functions": [
        "GetRenderHTML()",
        "GetFileList()",
        "NewFile(filename)",
        "GetFileContent(filename)"
    ]
});

你可以把这些保存为一个.js文件，预先定义好自己经常需要用到的短语。这就是LanEditor最简单的插件了。需要用的时候就把它们引入页面。
