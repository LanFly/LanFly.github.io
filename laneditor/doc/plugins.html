插件模式

LanEditor可以配置为插件模式

什么是插件模式？

有时候可能你只需要一个编辑器，而不需要ESC的菜单和多文件管理，也不需要自动保存，然后把它嵌入进已有的一个系统中。比如替换Wordpress后台的文章编辑器。目前我就是替换Wordpress的编辑器，然后编写一个LanEditor插件，这样Wordpress就变成了markdown文章了。

插件模式的ESC菜单功能失效，也就是LanEditor将不会监听ESC键。同时文件管理也不能用，不会自动保存，自动渲染也会关闭。但是符号补全和关键字提示可以使用提供的API选择开启或是关闭。

这样可以不干扰某些场合下的使用体验。

开启插件模式

可以在初始化的时候提供PluginsMode参数开启插件模式：

<script type="text/javascript">
    $(document).ready(function(){
        //初始化 开启插件模式
        var lan = LanEditor.init({
            textelem: "editor",
            PluginsMode: true
        });
        //如果初始化失败，则显示出错信息
        if(lan.status == false){
            alter(lan.message);
            return ;
        }
    });
</script>

可选参数

开启插件模式后，你只能使用API来控制其他选项，下面是全部可用的选项：

  1. OpenSKL

    true/false 开启或关闭关键字提示

  2. OpenSKLAni

    true/false 开启或关闭关键字提示框动画

  3. OpenAutoSymbol

    true/false 开启或关闭符号自动补全

上面这些参数都是可选的，默认为true。

注意

插件模式下依然会自动读取上次的配置选项。任何时候你都应该显式的给出参数，否则你使用的将是上一次的配置。可能上次的配置不是你想要的。

插件模式下你依然可以使用API提供的GetRenderHTML函数获取渲染后的HTML。
